  • 软链接和硬链接

每创建一个硬链接相当于对资源添加了一个指针，对任意指针所有的更改都会影响指向的文件。删除一个硬链接不会删除资源，但是如果删除的是资源的最后一个硬链接，那么这个文件就会彻底删除。

软链接（soft link）相当于创建了一个资源的快捷方式，更像是一种方便寻找资源的方式。删除指向资源的任意软链接不会对资源造成影响，但是如果资源被删除了，软链接就失效了。

  • 进程间通信方式

进程间的通信方式我了解的有：管道、FIFO、信号量、消息队列、共享内存和嵌套字。

管道是将一个进程的输出作为另一个进程的输入，这个过程是单向的。并且只能在亲子进程之间使用。FIFO即First Input First Output，也是一种管道，但是这种管道没有仅亲子进程使用的限制。管道的通病是速度比较慢。

信号量semaphore是一个计数器，可以用来控制多个进程对共享资源的访问。信号量有两个操作：P(sv)和V(sv)。举个例子：进程A通过P操作获取信号量，信号量-1后变为0，A可以使用资源。这时如果B通过P要使用资源，发现信号量为0，那么进程B就要等待A用完并使用V操作释放信号量之后才能使用。

ipcs -q Inter-Process Communication查看消息队列。共享内存即进程间共享一部分内存，还有嵌套字socket，这三种具体的没有去了解。

  • 线程通信方式

volatile变量用共享内存的方式进行通信。wait()和notify()方法进行通信。

  • 线程和进程

进程是资源调度的最小单位，是程序运行的一个实例。每创建一个进程，需要给该进程分配地址空间、堆栈段、数据段等。分配好资源之后将进程送入进程调度器，等待调度器给其分配cpu时间，程序才开始执行。

线程是cpu调度的最小单元，线程本身是不带资源的。一个进程可以有很多个线程，多个线程可是实现并发操作。

  • 并发和并行

并发是在一段时间中，多个线程轮流执行。并行是指线程齐头并进，比如多核cpu同时处理多个任务。

  • 堆和栈的区别

程序的内存空间分配从上到下分别是：栈、堆、静态区、文字常量区和代码执行区。

栈是存放临时变量等的内存空间，不能指定大小，栈是从上往下增长的。堆是程序员malloc手动分配的内存空间，可以指定分配的大小。堆是从下往上增长的。

网络

  • tcp三次握手

tcp三次握手是客户端和服务器建立连接的过程。

第1⃣️次握手过程：客户端向服务器发送SYN=1（）、seq=x的数据包。相当于“8633呼叫塔台，客户端进入SYN_SENT状态”。第2⃣️次握手过程：服务器收到SYN包之后，回复客户端SYN=1，ACK=1，seq=y，ack=x+1数据包，相当于“塔台收到，请回答”的过程。服务器进入SYN_RCVD状态。第3⃣️次握手：最后客户端向服务器发送ACK=1，seq=x+1，ack=y+1的包，至此客户端和服务器之间的连接就建立起来了。

  • 为什么需要三次握手

如果只有一次握手，那么客户端不知道服务器有没有收到数据包。如果只有两次握手，那么服务器就会持续等待客户端的数据，如果客户端一直不发送数据，服务器还处于继续等待的状态，会造成服务器的资源浪费。

  • tcp四次挥手

tcp四次挥手是客户端和服务器断开连接的过程。

第1⃣️次挥手：客户端主动关闭连接向服务器发送FIN=1，seq=x的包，此时客户端进入FIN_WAIT状态。第2⃣️次挥手：服务器收到FIN包建立连接状态，向服务发送ACK=1，seq=y，ack=x+1的包，进入CLOSE_WAIT状态。第3⃣️次挥手：服务器将剩余所有的数据都传给客户端之后，被动关闭，并向客户端发送FIN=1，seq=w，ack=x+1的FIN包。第4⃣️次挥手：客户端收到服务器的FIN包，向服务器回应ACK=1，seq=x+1，ack=w+1的ACK包，进入2MSL的TIME_WAIT状态之后变为CLOSE状态。服务器收到客户端的ACK请求，立马进入CLOSE状态。

  • http和https的区别

http（hyperlink text transfer protocol）超文本传输协议，是通过tcp/ip进行传输数据的网络协议。https是在http的基础上加了ssl安全验证的协议。

其区别是http是明文传输，有可能被不法分子抓到传输的数据。https加了ssl验证，传输的数据都通过密钥加密所以更加安全。

  • tcp、udp区别

tcp如果超时丢包了会重传更加稳定，另外数据一致性会得到保障所以更加安全。udp不考虑丢包的情况，传输的速度会更快。

  • tcp如何保证可靠

校验和、序列号、超时重传、流量控制、拥塞控制

  • tcp如何实现流量控制

服务器会维护一个滑动窗口，比如当前能接受的数据大小是x，服务器就会在下一个包中告诉客户端现在能接收到的数据最大是x，超过这个大小的数据就先不要传了。客户端收到之后，并不知道服务器数据的处理情况，所以会隔一段时间询问服务器当前滑动窗口的大小是多少。

  • DNS域名解析

DNS域名解析会先在本地，以此是浏览器、主机和路由器中查找是否有域名转为ip的缓存。如果依旧没有，将该域名依次转向运营商DNS缓存、根域名服务器、顶级服务器进行查找，直到找到目标域名的ip地址。然后将找到的结果存入缓存中。

  • http1和http2的区别

举个例子：如果通过http1请求一个带有css、js资源的html页面，页面加载过程会等先发送css的请求，然后服务器返回css资源，然后发送js请求，然后服务器返回js资源，直到所有资源均被客户端接收到之后再渲染页面。如果通过http2的话，请求css、js、html的过程是并发的，也就是css、js、html的请求会同时发出，不需要等上一个资源接收到之后再发送下一个资源的请求。

  • TIME_WATI状态的意义？

防止下次打开相同的tcp连接，会发送一个SYN请求，恰好上一个结束的ACK请求还没有发给服务器，服务器接收到两个请求，不知道该怎么办。

数据结构

  • 冒泡

如果有n个元素需要排序，第1⃣️次排序从第一个元素开始和其右边的元素进行比较，如果大于当前元素大于右边的元素，就和其交换位置。这样一轮排序下来就将最大的元素排到了最后面。第2⃣️次排序的时候继续从第一个元素开始和其右边的元素进行比较，这次只需要比较到n-1位，因为第一轮排序已经找到了最大的元素。继续第3、4、…、n-1次排序。

冒泡排序的平均时间复杂度是O(n^2)，最好的情况是刚好一次排序成功，时间复杂度是O(n)。

  • 快排

赋值基准元素pivot为第一个元素不变，然后有：index、left、right三个指针。index指向基准元素，left和right分别指向最左和最右的元素。

首先从right指针指向的元素开始，如果该元素大于pivot元素，那么right指针向左移动；如果小于pivot，那么把该元素填到index处，并将index调整到right指针所在的位置。

接下来看left指针，将left指针上的元素和pivot进行比较，如果小于pivot，left指针向右移动。如果大于pivot元素，那么将left上的元素填到index所在的位置，并将index指针调整到left所在的位置。

接着回到right指针，left指针。如此往返，直到left和right指针重合在一起。这个位置就是pivot元素应该填入的位置。

快排的平均时间复杂度是O(nlogn)，

  • 数组和链表的区别

数组的是栈分配的连续存储空间，查找起来比较方便，但是插入和删除比较复杂。

链表的元素有两个属性，一个是value，另一个是pointer指向下一个元素。链表是堆分配的，不要求存储空间的连续性。查找的复杂度是O(n)，插入和删除链表非常方便。非常适用于动态扩展。

数据库

  • 乐观锁、悲观锁

乐观锁：乐观估计一次事务不会发生错误，等该次事物提交之后验证数据的一致性。那么如何保证数据的一致性？CAS（Compare And Swap），在update之前先取没update之前的值，将update之后的值和实际值进行比较。但是也有问题，可以用version的方法避免。

悲观锁：悲观估计事务会发生错误，没有做修改之前就将记录加锁，等到修改结束之后再解除锁。

  • 事务ACID

Atomicity：Consistency、Isolation（事务之间是隔离的，相互的影响）、Durability

  • 索引

索引是一个数据结构，以一个单独的文件形式存在。可以加快数据库检索素速度。给某个数据加索引的方法有B+树或者哈希表。

  • 视图

视图是一种虚拟表，在不改变原数据的前提下，用不同的角度组合的方式观察数据表。创建、修改和删除视图对原数据表没有影响。

  • 三范式

一范式：每一列都是不可分割的原子数据

二范式：在1NF的基础上，每一列的数据都与主键相关

三范式：在2NF的基础上，每一列的数据都与主键直接相关。

  • 数据库隔离级别

read uncommitted，read committed，repeatable read，serializable

  • 关系型数据库和非关系型

菲关系型数据库仅存key-value。

  • 不考虑隔离性的结果

脏读：A查询数据，B同时修改数据。然后B回滚，A此时读到的数据就是空欢喜。

不可重复读：和脏读的区别是A做最后一次查询的时候，B已经commit了。而前几次查询都是在B没有commit的时候进行的。

幻读：A查询数据，B插入了几行数据，A查询结果会发现多出来几列。
